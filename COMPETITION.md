# WikiGen - AI 驱动的学习型网站生成工具

## 项目背景与灵感来源

WikiGen 的灵感来源于 **Claude.ai** 的网页展示界面，该界面以对话形式展示 AI 的思考过程与生成结果。我们从中得到启发：**能否将这种对话式的交互体验，转化为一个专门用于学习的网站生成工具？**

**Wiki** 取自 Wikipedia（维基百科），代表知识聚合与共享；**Gen** 取自 Generation（生成），象征 AI 的创造性产出。两者结合，寓意着通过 AI 对话生成知识型网站，让每个人都能轻松创建属于自己的学习资料库、知识笔记或教学页面。

## 设计理念

### 1. **聊天即工作区**
- **核心理念**：最自然的创作方式是与 AI 对话。用户无需掌握编程或设计技能，只需通过聊天描述需求，AI 即可理解并生成网站。
- **类比**：如同与一位资深网站设计师和开发者进行头脑风暴，用户提出想法，AI 负责实现。

### 2. **渐进式生成**
- 从简单描述开始，通过多轮对话逐步细化需求（布局、颜色、功能等）。
- 每一轮对话都会实时更新网站预览，形成“描述-生成-反馈-优化”的闭环。

### 3. **学习型导向**
- 生成的网站不仅美观，更注重内容的可读性、结构化和教育价值。
- 支持 Markdown 渲染、代码高亮、数学公式等学习常用元素。

### 4. **用户赋权**
- 让非技术用户也能拥有创建专业网站的能力。
- 全程透明，用户可以看到对话如何一步步转化为代码，甚至可导出完整的 HTML/CSS/JS 文件进行二次开发。

## 核心功能

### 💬 **对话式创作**
- 自然语言界面，支持中英文混合输入。
- 对话历史持久化，可随时回溯和修改之前的指令。
- 流式响应，模拟真人聊天体验。

### 🚀 **实时网站生成**
- 基于对话内容，实时生成符合语义的 HTML、CSS、JavaScript 代码。
- 响应式设计，自动适配桌面、平板、手机等不同屏幕。
- 内置预览器，支持多设备视图切换。

### 📁 **项目管理**
- 每个对话会话保存为一个项目，包含完整的聊天记录和生成的网站。
- 可视化仪表板，以卡片形式展示所有项目，支持搜索和筛选。
- 一键恢复会话，继续编辑或重新生成。

### 🎨 **现代化 UI/UX**
- 采用 UnoCSS 实现原子化 CSS，确保界面简洁、加载快速。
- 深色/浅色双主题，以石色（stone）为主色调，营造专注的学习氛围。
- 全程使用 Phosphor 图标，保持视觉统一性。

## 技术栈选型与理由

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| **Vue 3.5** | 3.5.x | 最新稳定版，Composition API 提供更好的逻辑复用与类型支持。 |
| **TypeScript** | ~5.9.3 | 增强代码可靠性，提供完善的类型提示，减少运行时错误。 |
| **Vite 8** | 8.x (beta) | 极速的开发服务器与构建工具，支持现代前端生态。 |
| **UnoCSS** | 66.6.6 | 原子化 CSS 引擎，Wind4 预设兼容 Tailwind，开发效率高。 |
| **Phosphor Icons** | 通过 UnoCSS 集成 | 图标风格统一，支持按需加载，减少打包体积。 |
| **Vue Router 4** | 4.5.0 | 官方路由解决方案，与 Vue 3 深度集成。 |
| **DeepSeek API** | 最新模型 | 国产优秀大模型，性价比高，支持长上下文与流式输出。 |
| **markstream-vue** | 0.0.8-beta.2 | 专为 Vue 设计的 Markdown 流式渲染组件，提升阅读体验。 |
| **PNPM** | 10.30.3 | 快速、磁盘空间高效的包管理器，适合现代 monorepo 项目。 |

**选型总结**：
- **前端框架**：Vue 3.5 + TypeScript 提供良好的开发体验与类型安全。
- **样式方案**：UnoCSS 原子化类名，结合 Attributify 与 Variant Group，书写简洁且性能优异。
- **构建工具**：Vite 8 带来更快的冷启动与热更新，适合快速迭代。
- **AI 集成**：DeepSeek API 作为国产模型，在中文场景下表现优异，成本可控。
- **包管理**：PNPM 确保依赖安装准确且节省磁盘空间。

## 架构设计

### 1. **模块分层**
```
┌─────────────────────────────────────────────┐
│                UI 层 (Components)           │
│   ChatInterface / WebsiteViewer / Dashboard │
├─────────────────────────────────────────────┤
│            逻辑层 (Composables)             │
│     useChat / useProject / useAI / useTheme │
├─────────────────────────────────────────────┤
│            数据层 (LocalStorage)            │
│        项目数据 / 对话历史 / 用户配置        │
├─────────────────────────────────────────────┤
│            服务层 (API Clients)             │
│          DeepSeek API / 本地存储服务         │
└─────────────────────────────────────────────┘
```

### 2. **状态管理**
- 使用 Vue Composition API 编写可复用的 `composable` 函数，管理聊天、项目、AI 请求等状态。
- 状态持久化通过 `localStorage` 实现，保证刷新后数据不丢失。

### 3. **AI 提示工程**
- 所有 AI 提示词（prompt）独立存放在 `src/prompts/` 目录下，以 TypeScript 文件导出。
- 提示词结构清晰，包含系统指令、用户示例、输出格式约束等，便于迭代优化。

### 4. **响应式与性能**
- 采用移动优先的响应式设计，使用 UnoCSS 的断点系统（sm、md、lg）适配不同设备。
- 代码分割与懒加载路由，减少首屏加载时间。
- AI 响应流式传输，用户无需等待完整生成即可看到部分结果。

## 创新点

1. **对话驱动的网站生成**
   - 将复杂的网站开发流程简化为自然语言对话，降低使用门槛。

2. **实时双向反馈**
   - 用户在聊天框中输入需求，右侧预览区域实时更新，形成即时反馈循环。

3. **学习型内容优化**
   - AI 不仅生成布局与样式，还会根据主题自动补充相关内容（如术语解释、参考链接等），增强网站的教育价值。

4. **全链路可追溯**
   - 每个网站的生成过程（对话记录、中间版本）都被完整保存，方便复盘与分享。

5. **国产技术栈**
   - 采用 DeepSeek 作为 AI 引擎，结合 Vue、UnoCSS 等国内开发者喜爱的技术，体现国产开源生态的成熟度。

## 未来展望

### 短期规划
- **多 AI 代理协作**：引入不同角色的 AI（设计师、前端工程师、内容专家），提供更专业的生成结果。
- **模板库**：内置常见学习场景模板（课堂笔记、技术文档、个人博客等），用户可基于模板快速开始。
- **导出增强**：支持导出为静态网站、PDF 或 Word 文档，方便离线阅读与打印。

### 中长期规划
- **插件系统**：允许开发者编写自定义插件，扩展 AI 的能力（如接入第三方 API、添加特殊组件）。
- **团队协作**：支持多人同时编辑同一项目，实现远程学习小组共同创作。
- **AI 训练数据反馈**：将用户对生成结果的评分反馈给模型，持续优化生成质量。

## 总结

WikiGen 是一款以 **对话为核心**、**生成为目标**、**学习为导向** 的 AI 驱动工具。它不仅仅是又一个“低代码”平台，而是试图重新定义人与机器在创作过程中的关系——让 AI 成为你的创作伙伴，而非仅仅执行命令的工具。

我们相信，通过降低技术门槛，让更多人能够轻松创建高质量的学习资料网站，WikiGen 将为个人学习、知识共享乃至在线教育带来新的可能性。
